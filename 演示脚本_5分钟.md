# EvolveFlow 使用讲解稿

这份稿子可以边操作边讲。每一段前面的「操作」只给自己看，真正开口时按「讲解」里的话自然说就行。

## 1. 开始

操作：打开 PowerShell，进入项目目录，启动桌面端。

```powershell
cd "C:\Users\52376\Desktop\课程与学习\软件工程\evolveflow"
npm run tauri -w @evolveflow/desktop-tauri -- dev
```

讲解：

大家好，我现在打开的是 EvolveFlow。它是一个面向学生日常学习的任务和日程管理工具，核心思路不是只做一个待办列表，而是把任务、日历、自动排程、学习数据分析和 AI Agent 放在同一套工作流里。

平时学生要处理的事情其实挺碎的，比如期末复习、数据库实验、计算机组成作业、英语六级练习，还有一些固定课程和考试时间。如果这些东西分散在不同软件里，很容易出现两个问题：一个是任务记了但没有安排时间，另一个是安排了时间但执行之后没有复盘。EvolveFlow 想解决的就是这个闭环问题。

我会按照真实使用顺序来走一遍：先看今天要做什么，再管理任务池，然后看日历排程，接着使用 AI Agent，最后再看命令行入口。

## 2. 今日页

操作：进入左侧「今日」页面。

讲解：

打开应用后，我一般会先看「今日」页。这里相当于是一天的学习工作台。

顶部这些卡片会汇总今天的基本情况，比如今天有多少任务、多少固定事件、预计专注时间、完成度和提醒数量。这样我不用先翻任务列表，就能知道今天大概忙不忙。

中间这里是今日计划，如果系统已经排过程，就会按时间显示今天每个时间段做什么。比如某个时间段复习数据库，某个时间段写计算机组成作业，或者晚上安排英语六级练习。固定事件和学习任务会放在同一个视图里，所以我能直接看出今天有没有时间冲突。

右边是 AI 助手和提醒区域。这里不会替代完整对话，但可以给一个快速提示，比如今天优先关注什么、还有哪些提醒没处理。

## 3. 快速添加任务

操作：在今日页快速添加输入框里输入一个正常学习任务，例如：

```text
整理数据库事务和并发控制错题
```

点击「添加」。

讲解：

如果我临时想到一个任务，可以直接在今日页快速添加。比如现在我添加一个“整理数据库事务和并发控制错题”。这种任务不需要一开始就填完整表单，先记下来就行。

添加之后，它会进入任务池，也会在今日任务里出现。后面如果我需要补充截止日期、预计时长、所属项目或者标签，再去任务页详细编辑。

这个设计对学生比较实用，因为很多任务都是临时想到的。先记录下来，避免忘掉；之后再整理优先级和时间安排。

## 4. 自动排程

操作：点击今日页的「自动排程」或「自动规划」。

讲解：

任务记下来以后，下一步就是把任务安排到具体时间里。这里可以点击自动排程。

系统会结合几个因素来安排：任务预计时长、截止日期、当天已有事件、工作时间偏好，以及已经锁定的时间块。比如固定课程不能被覆盖，已经锁定的复习时间也不能随便移动。

排程完成后，任务就不只是一个待办标题，而是落到了具体时间段里。这一点很重要，因为学生每天真正缺的通常不是任务列表，而是“什么时候做、先做哪个、能不能做完”的安排。

如果某个时间块已经确定，比如上课、考试、自习预约，就可以锁定。以后自动排程时，系统会尊重这些锁定块。

## 5. 任务页

操作：进入左侧「任务」页面。

讲解：

接下来进入「任务」页。这里是完整任务池，所有学习任务都会集中在这里管理。

顶部可以看到任务统计，比如总任务数、待办数量、已完成数量、延期数量和锁定数量。下面是快速添加和详细创建。

快速添加适合记录简单任务，比如“背六级高频词”或者“完成计算机组成作业”。如果任务比较复杂，就用详细创建。

详细创建里可以补充更多信息：任务属于哪门课，什么时候截止，预计需要多长时间，有哪些标签，是否需要拆成子任务。比如“准备数据库期末复习”这个任务，就可以拆成 SQL 练习、事务与并发控制、索引复习、实验报告整理几个子任务。

任务页还支持按项目、标签和状态筛选。比如我只想看数据库相关任务，就可以按项目筛选；只想看期末复习相关内容，就可以按标签筛选。这样任务多的时候不会乱。

每条任务右侧有一些操作按钮。可以完成任务、延期任务、取消任务、锁定任务、编辑任务或者删除任务。对于已经安排好时间的任务，我会倾向于锁定，避免后续排程把它打乱。

## 6. 日历页

操作：进入左侧「日历」页面，切换日视图、周视图、月视图。

讲解：

任务页解决的是“有什么事”，日历页解决的是“什么时候做”。

这里可以切换日视图、周视图和月视图。日视图适合看今天每个小时怎么安排；周视图适合看一周的学习强度；月视图适合看考试、提交和长期复习节奏。

如果有固定事件，比如数据库实验课、线下自习、考试时间，可以点击「添加事件」。事件保存后会显示在日历里，并参与排程判断。

日历里的时间块也可以调整。如果某个学习任务安排得不合适，可以把它拖到其他时间段。固定课程或者重要安排可以锁定，防止自动排程覆盖。

这样任务和日历不是割裂的：任务池里的学习目标会变成时间块，时间块执行后又会影响完成情况和后面的分析。

## 7. AI 页

操作：进入左侧「AI」页面，确认顶部状态显示 DeepSeek 和 deepseek-v4-flash。

讲解：

现在进入 AI 页。这里的 AI 产品固定使用 DeepSeek-V4-Flash，服务商是 DeepSeek，模型是 deepseek-v4-flash。它不是多模型选择器，而是直接作为 EvolveFlow 的内置 Agent 来使用。

如果设置页已经保存了 API Key，这里会显示连接正常。AI 的作用不是简单聊天，而是结合本地任务、日程和完成情况来给建议。

比如我可以问：

```text
我今天有哪些学习任务？请按紧急程度帮我排序。
```

这个问题会让 Agent 读取今天的任务和日程，然后给出更具体的建议。它应该能识别数据库、计算机组成、英语六级这些任务，而不是只回答一些泛泛的学习建议。

## 8. AI 模式

操作：在 AI 页顶部切换 Chat、Plan、Auto、YOLO。

讲解：

AI 页有四种模式。

Chat 是普通对话模式，适合问问题和查信息，不会主动修改数据。

Plan 是规划模式，适合让它帮我制定学习顺序或者复习计划，但它不会执行写入操作。

Auto 是自动模式。这个模式下，Agent 可以调用工具，比如读取任务、创建任务、调整任务。遇到写入操作时，页面会出现确认卡片，我确认之后它才会执行。

YOLO 是信任模式，会跳过逐项确认。这个模式只适合我非常确定要让 Agent 直接执行的时候使用。

日常用法上，我一般先用 Chat 或 Plan 问建议。如果我确实想让 Agent 帮我创建任务或整理任务，再切到 Auto。这样既能用到 Agent 能力，又不会让它在我不知道的情况下修改数据。

## 9. 让 AI 帮忙整理任务

操作：保持 AI 页，切换到 Auto，输入：

```text
帮我创建三个任务：复习数据库事务、完成计算机组成作业、背六级高频词，每个任务预计 45 分钟。
```

如果出现确认卡片，点击确认。

讲解：

比如我现在让 Agent 帮我创建几个学习任务。它会理解自然语言，把里面的任务拆出来，再准备调用任务创建工具。

因为这是写入操作，所以 Auto 模式会要求确认。确认卡片里会显示它准备调用的工具和参数摘要。确认后任务才会真正写入本地数据库。

这就相当于把自然语言输入和本地任务系统连起来了。用户不用自己一个个填表，也不用担心 AI 无声无息地修改数据。

创建完成后，我可以回到任务页刷新，就能看到这些任务已经进入任务池。

## 10. 分析页

操作：进入左侧「分析」页面，切换概览、项目分析、时间分析等区域。

讲解：

当任务使用一段时间后，就可以进入分析页复盘。

这里可以看到任务总量、完成数量、待办数量、完成率，以及不同项目的任务分布。对学生来说，项目通常就对应不同课程，比如数据库、计算机组成、英语六级、高等数学。

如果某一门课任务很多、完成率低，就说明这门课最近压力比较大。后续排程时，就可以给它更多时间。

时间分析则帮助判断自己在哪些时段更容易完成任务。比如晚上适合做英语练习，白天更适合写实验报告或复习数据库，这些都可以通过数据慢慢调整。

所以分析页不是单独的统计面板，而是反过来服务后续计划。

## 11. 设置页

操作：进入左侧「设置」页面。

讲解：

设置页主要管理几个基础偏好。

首先是主题和语言，可以切换浅色、深色以及中英文。

然后是工作时间，比如设置从早上九点到晚上十点。自动排程会参考这个范围，不会默认把任务排到不合适的时间。

提醒偏好可以设置提前几分钟提醒，也可以选择排程风格。宽松模式会留更多空隙，平衡模式比较适合日常使用，紧凑模式会尽量充分利用时间。

AI 配置这里固定显示 DeepSeek 和 deepseek-v4-flash，只需要保存 DeepSeek API Key，不需要选择其他服务商或模型。

下面还有备份区域。因为数据是本地优先保存的，做大量改动前可以备份一次，后续需要时再恢复。

## 12. CLI 入口

操作：打开一个新的 PowerShell，直接运行：

```powershell
evolveflow
```

也可以运行短别名：

```powershell
eflow
```

讲解：

除了桌面端，EvolveFlow 还有命令行入口。现在不需要输入项目里的长路径，直接在 PowerShell 里输入 `evolveflow` 就能进入 Agent。

进入后会看到当前服务商、模型、模式和 API Key 状态：

```text
Provider: DeepSeek
Model: deepseek-v4-flash
Mode: Chat
```

在这个界面里，可以直接输入自然语言问题，比如：

```text
我今天有什么任务？
```

也可以输入内部命令：

```text
/status
```

查看连接状态。

```text
/mode auto
```

切换到 Auto 模式。

```text
/exit
```

退出 CLI Agent。

这个入口和桌面端共享同一套 Agent Core，也共享同一份本地数据库。所以在 CLI 里创建的任务，回到桌面端任务页也能看到。

## 13. CLI 常用命令

操作：在 PowerShell 里运行这些命令。

```powershell
evolveflow --help
```

讲解：

`--help` 可以查看所有命令。CLI 主要支持任务、事件、排程、历史记录和 AI 对话。

查看任务：

```powershell
evolveflow task list
```

创建任务：

```powershell
evolveflow task create -t "准备六级听力练习" --duration 30 --project "英语六级" --tags "听力,练习"
```

自动排程：

```powershell
evolveflow schedule plan --date 2026-06-09
```

单次向 Agent 提问：

```powershell
evolveflow -p "我今天有什么任务？"
```

用 Plan 模式规划：

```powershell
evolveflow --mode plan -p "帮我规划今晚两个小时的复习顺序。"
```

讲解：

这些命令适合终端用户或者需要快速操作的时候使用。比如我不打开桌面端，也可以直接查看任务、创建任务、进行排程，或者向 Agent 提问。

这就让 EvolveFlow 有两种入口：桌面端适合可视化管理，CLI 适合快速操作和 Agent 对话。

## 14. 收尾

操作：回到桌面端「今日」或「任务」页面。

讲解：

整体看下来，EvolveFlow 的核心不是单个页面，而是一套学习管理流程。

任务页负责把学习目标记录清楚，日历页负责把目标安排到时间里，今日页负责每天执行，AI Agent 负责基于本地数据给建议和调用工具，分析页负责复盘完成情况，CLI 则提供另一种更直接的使用入口。

对学生来说，这套流程可以覆盖从“我有哪些任务”，到“我今天怎么安排”，再到“完成得怎么样”的整个过程。它不是只记录待办，而是把待办变成可以执行、可以调整、可以复盘的学习计划。

## 15. 如果命令不能直接运行

操作：如果 PowerShell 里提示找不到 `evolveflow`，回到项目目录运行：

```powershell
cd "C:\Users\52376\Desktop\课程与学习\软件工程\evolveflow"
npm run build -w @evolveflow/cli
npm link -w @evolveflow/cli
```

讲解：

如果换了一台电脑，或者 CLI 还没有链接到本机环境，需要先构建并链接一次。链接成功后，就可以在 PowerShell 里直接使用 `evolveflow` 或 `eflow`。
